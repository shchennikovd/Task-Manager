# 📝 DevFlow AI (Task Manager)

# 📖 Описание

**DevFlow AI** - это продвинутое Full-Stack приложение для управления задачами, спроектированное специально для разработчиков. Проект объединяет классические методологии управления временем (Входящие, Сегодня, Календарь) с интегрированным AI-ассистентом. 

Проект построен по современной микросервисной/клиент-серверной архитектуре: 
- **Frontend** - SPA на React, обеспечивающее мгновенный отклик интерфейса.
- **Backend** - надежный REST API сервис на базе Spring Boot (Java 17) с контейнеризированной базой данных. 

Основная цель приложения - дать возможность не просто вести списки дел, но и получать интеллектуальную контекстную помощь по конкретным задачам прямо в интерфейсе приложения.

---

# ✨ Возможности

- **Продвинутое управление задачами:** Создание, редактирование, удаление и смена статусов. Поддержка приоритетов, дат выполнения и цветового кодирования.
- **Интеллектуальный ассистент:** Встроенный AI-чат с сохранением истории. Позволяет отправить контекст конкретной задачи нейросети в один клик.
- **Различные представления:**
  - *Входящие:* Обзор предстоящих и просроченных задач.
  - *Сегодня:* Сводка задач на день с визуализацией прогресса.
  - *Календарь:* Интерактивная сетка на месяц с индикацией загруженности.
- **Глобальный поиск:** Быстрый поиск задач с помощью Command Palette (шорткат `⌘K` / `Ctrl+K`).
- **Безопасная авторизация:** JWT-аутентификация с жестким разграничением доступа (пользователь может взаимодействовать только со своими проектами и задачами).

# 🏗 Архитектура

Архитектура проекта построена вокруг принципов разделения ответственности (Separation of Concerns).

**Frontend-слой:**
1. **Сетевое взаимодействие (Network Layer):** Централизованный API-клиент инкапсулирует работу с JWT-токенами, заголовками и перехватывает ошибки (например, глобальная обработка 401 Unauthorized).
2. **Управление данными (Data Layer):** Реализовано через самописный паттерн *Observer* (`TaskStore`, `ChatStore`), что делает бизнес-логику независимой от UI-фреймворка.

**Backend-слой:**
1. **REST API (Spring Boot):** Обработка бизнес-логики, валидация данных и связь с AI.
2. **Безопасность:** Идентификация сессий через Bearer токены. Доступ к сущностям проверяется на уровне репозитория (например, `findBy...AndOwner(..., user)`), исключая возможность IDOR-уязвимостей.
3. **Изолированная БД:** База данных поднимается в изолированном Docker-контейнере, что исключает конфликты окружений у разных разработчиков.

# 🚀 Инструкция запуска

Для полноценной работы приложения требуется запустить как Backend, так и Frontend.

### Предварительные требования
- Установленный **Docker Desktop** (для запуска БД и бэкенда).
- Установленный **Node.js** (v18+).

### 1. Запуск Backend (API + База данных)
Перейдите в папку бэкенда (убедитесь, что находитесь в актуальной ветке, например, `dev/backend`):
```bash
cd backend
docker compose up --build
```
*Docker автоматически поднимет базу данных (PostgreSQL) и Spring Boot сервер.*
- Backend будет доступен по адресу: `http://localhost:8080`
- Проверка работоспособности: перейдите по `http://localhost:8080/api/health` (сервер должен ответить `Backend is working`).

### 2. Запуск Frontend
Откройте новый терминал и перейдите в папку фронтенда:
```bash
cd frontend
npm install
npm run dev
```
*Frontend будет доступен по адресу `http://localhost:5173`. В `vite.config.ts` уже настроен proxy: все запросы на `/api` автоматически маршрутизируются на `http://localhost:8080`.*

# 🔌 Краткая спецификация API

Взаимодействие клиента и сервера происходит через REST API. Все защищенные эндпоинты требуют передачи заголовка: `Authorization: Bearer <accessToken>`.

- `POST /api/auth/signup` - Регистрация (принимает `name`, `email`, `password`).
- `POST /api/auth/login` - Авторизация (возвращает `accessToken` и объект `user`).
- `GET /api/auth/me` - Получение данных текущего пользователя.
- `GET, POST, PATCH, DELETE /api/projects` - CRUD операции над проектами/задачами пользователя.
